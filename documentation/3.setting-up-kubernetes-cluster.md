# Talos Kubernetes on Proxmox — OpenTofu

Full IaC stack to provision a Talos 1.12.3 Kubernetes cluster on Proxmox `pve-node2`
with Cilium as the CNI. One control plane (`talos-cp`) + one worker (`talos-worker1`).

---

## Architecture Overview

```
Proxmox pve-node2
├── talos-cp      (VM 200)  — Talos control plane, etcd, API server
└── talos-worker1 (VM 201)  — Talos worker

CNI:       Cilium (kube-proxy replacement via eBPF, flannel disabled)
State:     Local OpenTofu state (migrate to MinIO/S3 for production)
Secrets:   Stored in tofu state — protect the state file
```

## Prerequisites

### Tools (install on your workstation)

```bash
# OpenTofu
curl -fsSL https://get.opentofu.org/install-opentofu.sh | bash

# talosctl
curl -sL https://talos.dev/install | bash

# kubectl
# https://kubernetes.io/docs/tasks/tools/

# Verify
tofu version   # >= 1.6
talosctl version --client
```

### Networking pre-check

Before running `tofu apply`, the IPs you assign in `terraform.tfvars` **must be routable**
and the VMs must be able to reach them on first boot. The recommended approach is:

- **pfSense DHCP reservation** — get the VM MAC from Proxmox after `tofu apply` creates the VM,
  add a reservation, then restart the VM. (Two-phase approach.)
- **OR** — reserve the IPs before creation by pre-configuring static DHCP leases keyed on VM ID.

The cleanest homelab approach: create the VMs first (`tofu apply -target=proxmox_virtual_environment_vm.talos_cp`),
note the MAC, add pfSense reservations, then run full `tofu apply`.

## Step 1 — Create the Proxmox API User

Do this **once** on your Proxmox host (SSH in as root or use the Shell in the UI).

### 1a. Create the user and role

```bash
# Create a dedicated role with the minimum permissions needed
pveum role add OpenTofuRole -privs "
  Datastore.AllocateSpace
  Datastore.AllocateTemplate
  Datastore.Audit
  Pool.Audit
  SDN.Use
  Sys.Audit
  Sys.Modify
  VM.Allocate
  VM.Audit
  VM.Clone
  VM.Config.CDROM
  VM.Config.CPU
  VM.Config.Cloudinit
  VM.Config.Disk
  VM.Config.HWType
  VM.Config.Memory
  VM.Config.Network
  VM.Config.Options
  VM.Monitor
  VM.PowerMgmt
"

# Create the user (PVE realm, not PAM — no shell access)
pveum user add opentofu@pve --comment "OpenTofu automation user"

# Assign the role at the root path (covers all nodes and datastores)
pveum aclmod / -user opentofu@pve -role OpenTofuRole
```

### 1b. Create an API token

```bash
# Create token — output contains the secret, shown only once
pveum user token add opentofu@pve opentofu-token --privsep=0

# Example output:
# ┌──────────────┬──────────────────────────────────────┐
# │ key          │ value                                │
# ╞══════════════╪══════════════════════════════════════╡
# │ full-tokenid │ opentofu@pve!opentofu-token          │
# │ info         │ {"privsep":"0"}                      │
# │ value        │ xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx │
# └──────────────┴──────────────────────────────────────┘
```

Your `proxmox_api_token` variable value is:

```
opentofu@pve!opentofu-token=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

> **`privsep=0`** means the token inherits the full user permissions.
> If you want token-level permission scoping, omit `--privsep=0` and manage permissions per-token.

### 1c. Verify access

```bash
# From your workstation
curl -k -H "Authorization: PVEAPIToken=opentofu@pve!opentofu-token=<SECRET>" \
  https://<PROXMOX_IP>:8006/api2/json/nodes | python3 -m json.tool
```

You should see your node list. If you get a 403, recheck the ACL assignment.

## Step 2 — Configure OpenTofu

```bash
# Clone or copy this directory to your workstation

```

Edit `terraform.tfvars` with your real values (Proxmox IP, token, node IPs, etc.).


## Step 3 — Deploy

### 3a. Initialise providers

```bash
tofu init
```

### 3b. Review the plan

```bash
tofu plan
```

Check that it will create:
- 2 Proxmox VMs (`talos-cp`, `talos-worker1`)
- 1 `talos_machine_secrets` resource
- 2 `talos_machine_configuration_apply` resources
- 1 `talos_machine_bootstrap` resource
- 1 `helm_release` (Cilium)

### 3c. Apply — Phase 1 (VMs only)

If you need to note MACs for DHCP reservations, create the VMs first:

```bash
tofu apply \
  -target=proxmox_virtual_environment_vm.talos_cp \
  -target=proxmox_virtual_environment_vm.talos_worker
```

Then go to Proxmox UI → VM → Network → note the MAC. Add pfSense DHCP reservations.
Ensure the VMs can boot and get the IPs you assigned. Watch the console in Proxmox.

### 3d. Apply — Full stack

Once the VMs are reachable:

```bash
tofu apply
```

OpenTofu will:
1. Push machine configs to both nodes via Talos API (port 50000)
2. Talos installs itself to disk, reboots
3. Bootstrap etcd on the control plane
4. Wait for the Kubernetes API to come up
5. Pull kubeconfig
6. Install Cilium via Helm

> **Expected time:** 5–15 minutes depending on your network and storage speed.

## Step 4 — Access the Cluster

```bash
# Save credentials
tofu output -raw talosconfig > talosconfig
tofu output -raw kubeconfig  > kubeconfig

# Verify Talos health
export TALOSCONFIG=./talosconfig
talosctl health --nodes <CP_IP>

# Verify Kubernetes
export KUBECONFIG=./kubeconfig
kubectl get nodes -o wide

# Expected output — nodes should be Ready once Cilium is running:
# NAME            STATUS   ROLES           AGE    VERSION
# talos-cp        Ready    control-plane   5m     v1.32.x
# talos-worker1   Ready    <none>          4m     v1.32.x

# Verify Cilium
kubectl -n kube-system rollout status daemonset/cilium
kubectl -n kube-system exec ds/cilium -- cilium status
```

## Ongoing Operations

### Apply a config change to a running node

```bash
# Edit the patch file, then:
tofu apply

# talosctl directly (without tofu):
talosctl apply-config --nodes <IP> --file <config.yaml> --mode=reboot
```

### Upgrade Talos

```bash
talosctl upgrade --nodes <IP> \
  --image factory.talos.dev/installer/ce4c980550dd2ab1b17bbf2b08801c7eb59418eafe8f279833297925d67c7515:<NEW_VERSION>
```

### Upgrade Kubernetes

```bash
talosctl upgrade-k8s --nodes <CP_IP> --to <NEW_K8S_VERSION>
```

### Dashboard

```bash
talosctl dashboard --nodes <IP>
```

## Gotchas & Tips

| Issue | Resolution |
|---|---|
| `tofu apply` hangs at `talos_machine_configuration_apply` | VM hasn't booted yet or can't reach port 50000. Check Proxmox console, verify IP is reachable. |
| Nodes stuck in `NotReady` | Cilium isn't running yet. Check `kubectl -n kube-system get pods`. Give it 2–3 minutes. |
| Cilium pods in `Init:CrashLoopBackOff` | Missing kernel modules. Verify your Talos factory image includes the required extensions. |
| `etcd` fails to start | Clock skew between nodes. Ensure both VMs sync to NTP before joining. |
| Proxmox API 403 error | Recheck ACL: `pveum aclmod / -user opentofu@pve -role OpenTofuRole` |
| Lost state file | Secrets are gone. You'll need to wipe and re-provision the cluster. Keep state backups. |
