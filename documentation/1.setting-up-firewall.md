# pfSense Firewall Setup Guide

## 1. Introduction

This document describes how I set up pfSense on a **Rouafwit Mini PC Firewall N150** as the primary firewall and router for my homelab.

The goal of this guide is to provide a clear, repeatable reference for:

- Installing pfSense on the hardware.
- Assigning and planning network interfaces.
- Applying basic security best practices for day-to-day management.
- Capturing the current network design so future expansion remains consistent.

## 2. Installation Process

### 2.1 Downloading the Installation Media

Use the official Netgate process to download the pfSense installer image:

- Official guide: https://docs.netgate.com/pfsense/en/latest/install/download-installer-image.html

Recommended checklist:

1. Visit the official download page from the guide above.
2. Select the pfSense Community Edition installer (or Plus, depending on license).
3. Choose the correct architecture and image format for USB installation.
4. Download the image and checksum/signature files if available.
5. Verify the downloaded image integrity before writing it to USB.

### 2.2 Preparing Installation Media

Create a bootable USB installer from a laptop/desktop:

1. Insert a USB drive (minimum 1-2 GB, all data on it will be erased).
2. Use a disk imaging tool to write the pfSense image to USB (for example, balenaEtcher, Rufus, or `dd` on Linux/macOS).
3. Safely eject the USB after flashing completes.
4. Label the drive for reuse in recovery or reinstallation workflows.

### 2.3 pfSense Installation Walkthrough (Mini PC)

Follow the official installation flow:

- Official walkthrough: https://docs.netgate.com/pfsense/en/latest/install/install-walkthrough.html

High-level procedure used on the Rouafwit N150:

1. Connect keyboard/monitor to the Mini PC and insert the bootable USB.
2. Enter BIOS/UEFI and set USB as the first boot device.
3. Boot into the installer and start pfSense installation.
4. Accept defaults unless a custom disk layout is required.
5. Complete install, remove USB, and reboot into pfSense.
6. Perform initial console setup and assign interfaces.

## 3. Network Interface Configuration

### 3.1 WAN Configuration

- The **first network port** on the Rouafwit N150 is configured as **WAN**.
- This WAN port connects upstream to the ISP modem/ONT.

### 3.2 Primary LAN Configuration

- The **second network port** is configured as the **primary LAN**.
- This LAN port is connected to a switch that serves wired and downstream network segments.

### 3.3 Additional LAN for Access Point Router

- A separate LAN network is configured for a ** TPlink router**.
- The router is used in **Access Point mode** to provide home Wi-Fi.
- DHCP and routing are centralized on pfSense; the AP is used only for wireless bridging.

### 3.4 Planned Future LAN Port Usage

One additional LAN-capable port is reserved for future segmentation, with possible use cases:

- Dedicated network path for a NAS server.
- Dedicated DMZ segment for externally exposed services.

## 4. Security Enhancements

### 4.1 User Account Management

After initial setup, create a dedicated non-default administrative user from the pfSense UI.

Example account:

- Username: `my_user_name`

Suggested process:

1. Sign in to pfSense Web UI.
2. Navigate to **System -> User Manager**.
3. Add user `my_user_name` with a strong unique password.
4. Assign the required administrative privileges.
5. Test login with the new account.

### 4.2 Why Daily Operations Should Avoid `admin`

Use the new user account for daily management and keep the default `admin` account for restricted scenarios.

Security rationale:

- Reduces reliance on a well-known default username.
- Improves auditability of configuration changes (actions tied to named user).
- Preserves `admin` as an emergency **glassbreak** account if normal access fails.

Recommended policy:

- Daily tasks: use `my_user_name`.
- Emergency recovery only: use `admin account`.
